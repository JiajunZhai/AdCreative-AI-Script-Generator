# 产品需求文档 (PRD): AdCreative AI Script Generator

**版本**： v1.0
**状态**： 初稿
**目标**： 通过 AI 自动化生成基于买量逻辑的游戏素材脚本。

## 1. 项目概述

### 1.1 背景

当前移动游戏买量市场（UA）对素材消耗速度极快。创意团队面临“构思慢、爆款率不稳定、跨平台风格适配难”的问题。

### 1.2 目标

开发一款 AI 工具，用户输入游戏基础信息，工具自动产出符合营销心理学、分镜头精细、且具备效果预测的买量脚本。

## 2. 角色与流程

- **用户角色**： 游戏出海投放师、视频制作人、UA 经理。
- **核心流程**：
  1. 输入游戏元数据（标题/描述/商店链接）。
  2. 选择投放参数（平台/目标/风格）。
  3. AI 引擎匹配爆款模型。
  4. 输出结构化脚本及效果预测评分。

## 3. 功能需求 (Functional Requirements)

### 3.1 模块一：输入与解析 (Input Hub)

- **[F1.1] 文本录入**： 用户手动输入游戏标题、核心玩法、美术风格。
- **[F1.2] URL 抓取**： 输入 Google Play/App Store 链接，系统自动爬取描述、分类及评价，并进行摘要处理（利用 LLM 提取卖点）。
- **[F1.3] 投放参数选择**：
  - **目标平台**： TikTok (短平快), Facebook (高质内容), AdMob (试玩感), YouTube (横屏)。
  - **脚本语言**： 中、英、日、韩、泰等（支持一键翻译生成多语种版本）。

### 3.2 模块二：AI 创意引擎 (Core Engine)

- **[F2.1] 创意方向选择 (The "Angles")**：
  - 系统需内置至少 5 种买量逻辑：失败诱导、数值膨胀、沉浸式剧情、UGC评测、ASMR解压。
- **[F2.2] 动态 Prompt 组装**：
  - 后端根据用户输入的卖点，从“钩子库 (Hook Library)”中随机或匹配最优的黄金 3 秒方案。
- **[F2.3] 脚本结构化输出**：
  - 脚本需包含：场景序号、时间点、视觉画面描述、旁白文案、字幕提示、BGM建议、拍摄建议。

### 3.3 模块三：效果预测评分 (Prediction System)

- **[F3.1] 评分维度**：
  - **Hook Score (3秒吸睛度)**： 基于视觉冲突强度评分。
  - **Clarity Score (玩法易懂度)**： 评估新手在 10 秒内理解游戏的概率。
  - **Conversion Score (转化推力)**： 评估 CTA 强弱。
- **[F3.2] 竞品分析参考**： 提示当前方向与市场上哪类 Top 广告素材逻辑最接近。

### 3.4 模块四：脚本管理与导出

- **[F4.1] 在线编辑**： 允许用户手动微调 AI 生成的台词。
- **[F4.2] 多格式导出**： 支持 PDF、Excel (分镜头表格)、Word 导出。

## 4. 技术设计建议 (Technical Design)

### 4.1 技术栈建议

- **前端**： React.js + Tailwind CSS (保证响应式 UI)。
- **后端**： Python (FastAPI / Django) —— 便于处理 AI 逻辑。
- **AI 模型**：
  - LLM: GPT-4o 或 Claude 3.5 Sonnet (由于创意工作需要极高的逻辑连贯性)。
  - RAG (检索增强生成): 建议建立一个 Vector DB (向量数据库)，存储 Top 1000 热门广告案例的文字拆解，让 AI 生成前先“学习”当前的爆款结构。
- **爬虫**： Playstore Scraper API。

### 4.2 核心算法逻辑 (Prompt Logic)

```text
System Prompt: 
你是一位拥有10年经验的移动游戏买量专家。
任务: 基于用户输入的游戏卖点 {USP} 和目标平台 {Platform}，生成一个满足 AIDA 模型的脚本。
约束: 
1. 黄金3秒必须包含冲突；
2. 包含一个逻辑上的失败或成功闭环；
3. 输出格式必须为 JSON，包含场景描述和旁白。
```

## 5. UI/UX 设计要求

- **工作流引导**： 采用步骤条 (Stepper) 形式：输入 -> 设置 -> 生成 -> 预览。
- **分屏展示**： 左侧展示脚本文字，右侧展示对应的“创意心理学解析”或“效果预测雷达图”。
- **深色模式支持**： 适配视频制作人员长期使用的视觉习惯。

## 6. 非功能性需求 (Non-Functional Requirements)

- **生成速度**： AI 脚本生成应在 15-20 秒内完成。
- **准确性**： 避免 AI 幻觉产生不存在的游戏机制（需在 Prompt 中严格约束限制在用户输入范围内）。
- **安全性**： 用户上传的未发布游戏信息需加密存储。

## 7. 衡量成功的指标 (KPIs)

- **效率提升**： 平均每个脚本生成的耗时（目标：从人工 1 小时降至 AI 1 分钟）。
- **采纳率**： 用户生成脚本后，点击“导出”或“保存”的比例。
- **测试一致性**： AI 生成的脚本与实际投放素材的 CTR 关联度。

## 8. 路线图 (Roadmap)

- **MVP (v1.0)**: 实现基础文本录入、单一平台（TikTok）、3 种创意方向、Excel 导出。
- **v1.1**: 增加商店链接解析、多语种翻译、效果评分预测。
- **v2.0**: 增加 AI 配音预览（集成 ElevenLabs）、AI 自动生成草图 (DALL-E 3 / Midjourney) 指引视觉效果。

## 9. 全球本地化引擎 (Localization & Culture Intelligence)

### 9.1 核心地区买量风格画像 (Regional Creative DNA)

- **北美/欧洲 (T1 国家)**: 强调真实、幽默、个人英雄主义。真人出镜 (UGC)、失败诱导、强叙事性。避免过度“五毛特效”。
- **日本 (高 LTV 市场)**: 强调声优 (CV)、世界观、角色细节、沉浸感。垂直构图、强调排行榜口碑。禁忌生硬翻译。
- **东南亚 (高增长市场)**: 强调福利、社交、炫酷效果。直接诱导（满屏“Free 1000 Draws”），高饱和度配色，夸张真人表演。
- **中东 (大 R 市场)**: 强调权力、荣誉、部落合作。金色与沙漠色偏好，SLG 宏大场面。禁忌宗教敏感（RTL 排版）。

### 9.2 语言适配模块 (Localization Engine)

- **Transcreation (创译)**: 基于流行语重写，非机器直翻。
- **字体美学自动匹配**: 如日语可选毛笔体，欧美粗体 Meme 风。
- **UI 自动排版**: 针对德语超长词自动缩放，阿拉伯语自动镜像 (RTL) 排版。

### 9.3 竞品分析模块 (Competitor Intelligence)

- 实时抓取 [情报库] 进行视觉趋势和投放榜单分析，给出 [差异化建议]（例如：“目前该地区三消70%在做破屋翻新，建议尝试反转剧情”）。
